### 2. Git 版本控制與安全防護設定
* **任務內容**：設定 GitHub 遠端倉庫並進行初始化。
* **安全性防護 (重點)**：
  * 電子發票工具涉及極為敏感的資訊，包括：
    1. `credentials.json` (Google OAuth 2.0 用戶端識別碼)
    2. `service_account.json` (Google Sheet 服務帳戶私鑰)
    3. `token.json` (Gmail API 的永久授權 Token)
    4. `.env` (包含 Gemini/Anthropic API Key 等敏感字串)
  * 為防止這些密鑰不慎被上傳到公開的 GitHub 倉庫，我們第一時間建立了防範機制，撰寫 [`.gitignore`](file:///d:/AI%20Class%20ChenYu/AIClass/ReciptCheck/.gitignore)，確實遮蔽所有憑證檔案及本地下載暫存檔。
* **版本控制設定**：
  * 初始化本地 Git 庫（`git init`）。
  * 連結 GitHub 遠端倉庫位址：`https://github.com/ChenYuHsu413/PersonalReciptChecck.git`。
  * 根據使用者指令，已將本地 Git 帳戶的 `user.name` 綁定為 `ChenYuHsu413`。
